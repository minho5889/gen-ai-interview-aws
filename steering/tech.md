> **This is the most important steering file.** The original specs (now parked in `archive/`) were
> generated by a coding agent that *hallucinated* several Strands/AWS APIs. Everything below was verified against
> the actual packages installed in `.venv` (`strands_agents==1.42.0`, `strands_agents_tools==0.8.0`,
> `langgraph==1.2.4`, `mcp==1.27.2`, `opentelemetry==1.42.x`). When in doubt, read the source:
> `.venv/lib/python3.13/site-packages/strands/`.

## 0. Mental model: two layers, do not conflate them

| Layer | What it is | Examples |
|---|---|---|
| **Framework** (in-process) | The Python agent loop, tools, hooks, multi-agent topology | **Strands SDK**, LangGraph |
| **Platform** (managed AWS) | Where the agent *runs* and the services it leans on | **Bedrock AgentCore** (Runtime, Memory, Gateway, Identity, Observability, Code Interpreter, Browser), Bedrock model APIs, Bedrock Guardrails, Bedrock Knowledge Bases |

> **PE note:** A junior conflates "Strands" and "AgentCore." Strands is the *code* that decides
> what to do; AgentCore is the *substrate* (serverless microVM session isolation, managed memory,
> auth, an MCP gateway) that you deploy that code onto. Strands runs fine on plain Lambda/Fargate
> too — AgentCore buys you session isolation, long-running (up to 8h) execution, and managed memory
> without you building it. They compose; they are not alternatives.

## 1. Strands core — the real API (verified, v1.42.0)

```python
from strands import Agent, tool
from strands.models import BedrockModel

model = BedrockModel(
    # Model IDs are now cross-region inference profiles. The SDK's own docstring uses:
    model_id="global.anthropic.claude-sonnet-4-6",   # pick the current Sonnet/Haiku for your region
    region_name="us-east-1",
    temperature=0.2,
    max_tokens=2048,
    # Bedrock Guardrails wire in HERE, natively — you do not hand-roll output interception:
    guardrail_id="abc123",
    guardrail_version="DRAFT",
    guardrail_redact_input=True,
)

@tool
def get_order_status(order_id: int) -> str:
    """Return the status of an order. The docstring IS the tool description the model sees."""
    ...

agent = Agent(
    model=model,
    system_prompt="You are ...",
    tools=[get_order_status],
)
result = agent("Where is order 5?")   # synchronous; also: await agent.invoke_async(...)
```

**Verified `Agent(...)` constructor knobs you will actually use** (from `strands/agent/agent.py`):
`model`, `tools`, `system_prompt`, `hooks`, `session_manager`, `conversation_manager`,
`structured_output_model`, `callback_handler`, `trace_attributes`, `agent_id`, `name`,
`concurrent_invocation_mode`. These are real. Use them instead of reinventing them.

**Available model providers** (`strands/models/`): `bedrock`, `anthropic`, `gemini`, `sagemaker`,
`litellm`, `ollama`, `openai`, `mistral`, `writer`, `llamaapi`, `llamacpp`. So a local-dev story
(`OllamaModel`/`LiteLLMModel`) for sandbox testing is *supported by the SDK*, not a hack.

## 2. Multi-agent: pick the primitive on purpose

Strands gives you **four** ways to compose agents (`strands/multiagent/`). Choosing the wrong one
is the most common design smell in this repo's original specs.

| Primitive | Import / shape | Use when | Don't use when |
|---|---|---|---|
| **Agents-as-tools** | a `@tool` fn that internally builds + calls a sub-`Agent` | A parent needs an *occasional* specialist; the parent decides if/when to call it | You need guaranteed parallelism or a fixed DAG |
| **Graph** | `from strands.multiagent import GraphBuilder` → deterministic DAG of nodes | Fixed control flow, **parallel fan-out/fan-in**, conditional edges | Routing is genuinely open-ended |
| **Swarm** | `from strands.multiagent import Swarm` → autonomous handoff between peers | Agents should hand off to each other dynamically | You want determinism / auditability |
| **A2A** | `strands.multiagent.a2a` (+ `a2a_client` tool) | Cross-process / cross-service agents over the A2A protocol | Everything is in one process |

> **PE note:** "Run three reviewers in parallel and merge" is a **`GraphBuilder`** problem (parallel
> nodes → a join node), *not* "agents as tools." Agents-as-tools makes parallelism the model's
> decision, which is non-deterministic and unauditable. Reserve agents-as-tools for *optional*
> specialists. This is the #1 fix across the original specs.

## 3. Tools, MCP, and the prebuilt toolbox

- `strands_agents_tools` ships real, batteries-included tools (`.venv/.../strands_tools/`). The ones
  that matter here: `mcp_client`, `retrieve` (Bedrock Knowledge Bases), `agent_core_memory`
  (Bedrock AgentCore Memory), `mem0_memory`, `http_request`, `file_read`/`file_write`,
  `python_repl`, `code_interpreter`, `use_aws`, `a2a_client`, `handoff_to_user`, `calculator`.
- **MCP is consumed via `mcp_client`**, not by hand-writing JSON-RPC. An MCP server can be: a local
  `stdio` process (like this repo's `.mcp.json` servers), or a Lambda/Fargate-hosted HTTP MCP
  server, or surfaced through **AgentCore Gateway** (which turns existing Lambdas/APIs into MCP
  tools with managed auth).
- **Never let the model write SQL/commands.** Tools expose *typed, parameterized* operations. The
  `@tool` signature + docstring is the contract the model sees; the body uses prepared statements.

## 4. Memory & state — three distinct things people blur together

1. **Conversation/context window management** → `Agent(conversation_manager=...)`. Strands has
   sliding-window and summarizing conversation managers. This is how you prevent context blowout —
   not a hand-rolled "summarize old turns" loop.
2. **Durable session persistence (resumability)** → `Agent(session_manager=...)`. Real classes exist:
   `FileSessionManager`, `S3SessionManager`, `RepositorySessionManager` (`strands/session/`). This is
   the *correct* answer to "Lambda times out at 15 min, how do I resume the agent loop" — persist the
   session, rehydrate on the next invocation. You do **not** need to invent a DynamoDB scratchpad
   schema by hand; back a `RepositorySessionManager` with DynamoDB if you want that store.
3. **Long-term semantic/episodic memory** → the `agent_core_memory` or `mem0_memory` tools, or a
   vector store (Bedrock KB via `retrieve`, OpenSearch Serverless, or local FAISS for dev).

> **Embedding dimension gotcha:** Amazon **Titan Text Embeddings V2** outputs **1024** dims by
> default (configurable to 512/256). **1536** is Titan G1 / OpenAI `ada-002`. The original
> `agent_migration` design says "1536 (Titan v2)" — that's wrong and will silently mis-size your
> OpenSearch vector index.

## 5. Safety / guardrails — the idiomatic patterns

**Tool-call policy enforcement = a `HookProvider`, not an `@agent.on(...)` decorator** (that
decorator does not exist). Verified event: `BeforeToolCallEvent` *does* exist, with fields
`selected_tool`, `tool_use`, `invocation_state`, and **`cancel_tool`**.

```python
from strands.hooks import HookProvider, HookRegistry
from strands.hooks.events import BeforeToolCallEvent

class ReadOnlyGuard(HookProvider):
    BLOCKED = ("write", "delete", "update", "put_", "create", "drop")

    def register_hooks(self, registry: HookRegistry) -> None:
        registry.add_callback(BeforeToolCallEvent, self._enforce)

    def _enforce(self, event: BeforeToolCallEvent) -> None:
        name = event.tool_use["name"].lower()
        if any(verb in name for verb in self.BLOCKED):
            # Idiomatic: set cancel_tool, which returns a clean error tool-result to the model
            # so it can recover. Raising an exception kills the whole loop instead.
            event.cancel_tool = f"Blocked: '{name}' is a write op; this agent is read-only."

agent = Agent(model=model, tools=[...], hooks=[ReadOnlyGuard()])
```

> **PE note:** The original spec `raise`d a `SecurityException`. Prefer `event.cancel_tool = "..."`:
> it converts the violation into a tool-result the model sees and can route around, instead of
> crashing the run. Raise only for true panics. Defense in depth still applies: the hook is an
> *application* guard; the real boundary is the IAM role on the tool's compute (least privilege) and
> Bedrock Guardrails on the model.

**Model-level safety = Bedrock Guardrails**, configured directly on `BedrockModel` (see §1). PII
redaction, denied topics, prompt-injection/jailbreak filtering live there.

## 6. Observability — OTEL-native, not "mlflow autolog"

There is **no `mlflow.strands.autolog()`**. That call was hallucinated across the original specs.
The real mechanism (`strands/telemetry/config.py`):

```python
from strands.telemetry import StrandsTelemetry

StrandsTelemetry().setup_otlp_exporter().setup_console_exporter()
# Driven by standard OTEL env vars: OTEL_EXPORTER_OTLP_ENDPOINT, OTEL_EXPORTER_OTLP_HEADERS,
# OTEL_SERVICE_NAME. Strands emits spans for the agent loop, model calls, and tool calls.
```

From there, ship OTLP to: **AgentCore Observability** (managed, the on-AWS default), CloudWatch
(via the CloudWatch OTLP endpoint / ADOT collector), or any OTLP backend (Langfuse, Jaeger, etc.).
If you specifically want MLflow, MLflow ingests OTEL traces — wire MLflow as an OTLP backend; do
**not** call a `strands.autolog()` that doesn't exist.

## 7. Quick "hallucinated → real" cheat sheet (what to fix on sight)

| Original spec said | Reality (use this) |
|---|---|
| `mlflow.strands.autolog()` | `StrandsTelemetry().setup_otlp_exporter()` + OTEL env vars |
| `@agent.on(BeforeToolCallEvent)` decorator | `HookProvider` + `registry.add_callback(BeforeToolCallEvent, cb)`; set `event.cancel_tool` |
| `raise SecurityException(...)` to block a tool | `event.cancel_tool = "<reason>"` (clean, recoverable) |
| "subagents as tools, run in parallel" | `GraphBuilder` parallel nodes + join (deterministic) |
| "Titan Text Embeddings v2 → 1536 dims" | Titan V2 = **1024** (or 512/256) |
| Hand-rolled DynamoDB agent scratchpad for resume | `session_manager=` (`S3SessionManager` / `RepositorySessionManager`) |
| Bedrock output interception by hand | Bedrock Guardrails via `BedrockModel(guardrail_id=...)` |
| "Strands Evals" library | No verified package by that name — build evals with an LLM-as-judge harness or Bedrock model evaluation; **verify before naming a library in a spec** |
| `GEMINI_API_KEY` for a Bedrock system | Wrong provider; use AWS creds + `BedrockModel` (or `GeminiModel` *only* if you genuinely chose Gemini) |
| Claude 3.5 Sonnet hardcoded | Default to the current Claude (Sonnet 4.x / Haiku 4.5) via an inference-profile model ID |
